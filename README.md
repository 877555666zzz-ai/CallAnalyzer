# Call Analyzer — анализатор звонков (RU/KZ)

Сквозная система: **Сипуни → STT → LLM-анализ → PostgreSQL → дашборд + карточка Bitrix24**.
Из звонка делает готовые выводы («где деньги»), а не транскрипцию.

## Что уже работает (проверено end-to-end)
- **Ядро анализа** — транскрипт+метаданные → структурированный JSON (чеклист, ред-флаги
  отдельно по оператору/клиенту, классификация результата, причина отказа, метрики, сводка).
- **Метрики детерминированно** (talk ratio, паузы, реакция) — считает код, не LLM.
- **Правила без кода** — чеклист и запрещённые фразы в `configs/*.yaml` (вкл. «до/от 5 минут»).
- **БД** (SQLAlchemy, SQLite локально / PostgreSQL в проде) + маппинг менеджеров, несвязанные звонки.
- **Отчёт «где деньги»** — потери по менеджеру/этапу/причине, база vs менеджер, what-if прибыль, Telegram-текст.
- **Адаптер Сипуни** — выгрузка звонков, скачивание записей, операторы (подпись MD5 по их API).
- **Адаптер Bitrix24** — авто-создание полей карточки, запись результата/причины + комментарий в таймлайн.
- **STT-адаптер** — Whisper (стерео = диаризация по каналам) / Yandex / mock.
- **Оркестратор** — ingest→STT→анализ→БД→Bitrix, ретраи, несвязанные не теряются.
- **Веб-дашборд** — «где деньги», список звонков (поиск по номеру), drill-down (аудио + транскрипт
  с подсветкой ред-флагов + разбор).

## Быстрый старт (демо, без ключей)
```bash
pip install -r requirements.txt
python seed_and_report.py                 # наполнить демо-БД + отчёт «где деньги» в консоль
uvicorn dashboard.app:app --port 8000     # дашборд: http://localhost:8000
python src/pipeline.py --demo             # сквозной прогон одного звонка
```

## Боевой запуск
```bash
cp .env.example .env        # заполнить (см. ниже)
set -a; source .env; set +a
python run_production.py 01.06.2026 24.06.2026
uvicorn dashboard.app:app --host 0.0.0.0 --port 8000
```

## Что требуется от вас (минимум)
1. **Заполнить `.env`**: `SIPUNI_USER`/`SIPUNI_SECRET` (Настройки→API→Статистика), `BITRIX_WEBHOOK`
   (входящий вебхук), `ANTHROPIC_API_KEY`, при желании `DATABASE_URL` (PostgreSQL).
2. **Маппинг менеджеров**: внутренний номер Сипуни ↔ ФИО ↔ подразделение. Заполнить таблицу `manager`
   (через `store.upsert_manager(...)` или импортом `SipuniClient.operators()`). Без номера звонок
   уходит в `mapping_unmatched` — ничего не теряется.
3. **Экономика** в `configs/yandex_taxi_corp.yaml` → `economics`: подставить реальные `avg_deal_value`
   и `recovery_rate` (сейчас заглушки).
4. **Колонки CSV Сипуни**: один раз сверить заголовки выгрузки с `COLUMN_MAP` в `src/sipuni_client.py`.
5. **Включить STT** (когда будут боевые звонки): `pip install faster-whisper soundfile`, в `.env`
   `STT_MODE=whisper`. Финальный выбор движка — по бейк-офу на ваших стерео-звонках (§5 ТЗ).

Всё остальное (анализ, метрики, БД, отчёты, дашборд, запись в Bitrix) — автоматизировано.

## Структура
```
analysis_schema.json        контракт разбора (JSON Schema) — единая точка правды
configs/*.yaml              правила подразделения: чеклист, ред-флаги, экономика
src/analyzer.py             промпт → LLM → метрики → валидация
src/metrics.py              детерминированные метрики
src/llm_client.py           Anthropic / Mock
src/sipuni_client.py        ingest из Сипуни
src/bitrix_client.py        запись в карточку сделки
src/stt.py                  Whisper / Yandex / Mock
src/db.py, store.py         модель данных и запись
src/report_money.py         отчёт «где деньги» + Telegram-текст
src/pipeline.py             оркестратор
dashboard/                  FastAPI веб-дашборд
seed_and_report.py          демо-данные + отчёт
run_production.py           боевой запуск за период
```

## Этап 2 — что добавлено (готово к данным)
- **Конференсии** (`/conversions`): тёплый→закрытая, КП/КДЗ, физлица vs юрлица.
- **Скорость отработки тёплого лида**: время до первого звонка, конверсия по интервалам, по менеджерам.
- **Сверка с CRM** (`зона ответственности`): ловит, когда менеджер протащил физлицо в продажную стадию.
- **Топы для прослушивания** (`/tops`): проблемные / успешные / конфликтные / аномальные.
- **Ролевые виды**: `/boss` (руководитель) и `/rop` (РОП — соблюдение скрипта, эмпатия, ред-флаги).
- **Telegram** (`src/telegram_report.py`): сводка «где деньги» + алерты по ред-флагам. Запуск:
  `TELEGRAM_BOT_TOKEN=... TELEGRAM_CHAT_ID=... python -m src.telegram_report`
- **Шаринг записей** (`/share/{id}` → подписанная ссылка, `/r/{token}` → плеер): не публичные ссылки
  с истечением, лог доступа в `access_log` (кто слушал/копировал).
- **Синхронизация сделок из Bitrix** (`src/bitrix_sync.py`) — наполняет таблицу `deal` для конверсий.
- **Планировщик** (`scheduler.py`): автозабор новых звонков + вечерняя сводка в Telegram.

### Источники данных для Этапа 2
Конверсии/скорость/сверка считаются по таблице `deal`. В прод её наполняет `bitrix_sync.sync_deals`
(нужны кастомные поля воронки: метка «тёплый», время попадания в тёплые, время первого звонка —
см. SELECT в `bitrix_sync.py`). В демо таблица заполняется синтетикой из `seed_and_report.py`.
